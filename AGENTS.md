# Torpedo — Agent Guide

> **This file is the canonical guidance for any AI coding agent working in this repository.** `AGENTS.md` is the source of truth; `CLAUDE.md` is a symlink to it (so Claude Code auto-loads it while the portable `AGENTS.md` name serves every other agent tool). Edit `AGENTS.md`, never the symlink.

## What Torpedo is (and is not)

Torpedo is a **fork of [rayfish](https://github.com/rayfish/rayfish)**, a P2P mesh VPN powered by [iroh](https://iroh.computer). It is not original software; almost all architecture, protocol design, and code are upstream's. The fork exists for **one product goal**: make the overlay IPv4 subnet **configurable** so Torpedo can run alongside Tailscale on the same host. Upstream hardcodes `100.64.0.0/10` (the CGNAT range Tailscale also claims) and refuses to start next to Tailscale; Torpedo defaults to `10.88.0.0/16` and lets the operator pick any CIDR per network.

The binary is **`torpedo`**. The Cargo **package/library is still named `rayfish`** (`[package] name = "rayfish"`, `[[bin]] name = "torpedo"`), so all internal `use rayfish::…` paths, the `info,rayfish=debug` log filter, and the crate's own module names are deliberately unchanged — renaming the crate would be churn with no user-visible benefit and is out of scope.

## KEEP-ON-PURPOSE — do NOT rename these

The fork deliberately keeps a set of `rayfish` identifiers. These are load-bearing (they must match upstream infrastructure or would be pointless churn) and are protected by constraints in `spec/design_spec.py` + `reconcile.py`. **An agent must not "finish the rename" on any of these:**

- **Crate / library name `rayfish`** and every `use rayfish::…` path, plus the `info,rayfish=debug` tracing filter. Internal only; never user-visible.
- **Relay / discovery presets** — the `"rayfish"` config keyword and its preset URLs (`relay.iroh.rayfish.xyz`, `dns.iroh.rayfish.xyz`). Protected by **CON-001**; `reconcile.py`'s `relay_preset_untouched` check requires the literal `"rayfish" => Ok(preset.to_string())` to remain in `src/config.rs`. Renaming would point nodes at nonexistent infrastructure.
- **`REPO_SLUG = "rayfish/rayfish"`** in `src/update.rs` — the self-updater's GitHub target. The fork does not publish releases, so this stays pointed at upstream and the updater is disabled (see below).
- **Author attribution** (`dario@rayfish.xyz` in `Cargo.toml`) — upstream authorship is retained honestly.
- **The `.ray` Magic-DNS TLD** — the naming scheme is unchanged. (The resolver's IP itself is *not* a fixed value — see SUBNET-011: it is derived from the configured overlay subnet, e.g. `10.88.100.53` on the default `10.88.0.0/16`.)

Everything that *was* renamed (binary, ALPNs, listen port, config/log/socket paths, host DNS artifacts, deep-link scheme, launchd label, config-dir env var, DHT record names) is a **collision-prone host artifact or user-facing surface** that had to change so torpedo and rayfish can coexist on one machine. **CON-007** (`reconcile.py`'s `host_identity` check) is a curated-token regression gate that keeps those from creeping back in; it scans `src/**/*.rs` for a specific token list and must report `leak_count == 0`.

## Spec-first workflow (libspec + reconcile.py)

Changes in this fork are **spec-driven and committed one requirement at a time**:

- **`spec/design_spec.py`** — one Python class per requirement/constraint. The ID is the first line of the docstring (e.g. `SUBNET-001`, `RENAME-006`, `CON-007`). `Requirement` subclasses are structural/design (verified by reading the diff + code); `Constraint` subclasses carry an `enforcement_logic` Jinja expression and are the automatable gates. A `UserStory` captures intent.
- **`reconcile.py`** — the per-commit gate. Run `python3 reconcile.py` from the repo root; it must exit `0`. It runs eleven checks and prints a JSON context: `build` (cargo build), `clippy` (`-D warnings`, 0 warnings), `test` (cargo test), `grep_hardcoded_cgnat` (no stray `100.64`/`100.100.100` literals in `membership.rs`/`tun.rs`/`dns.rs` beyond allowed comment lines), `relay_preset_untouched` (**CON-001**, must equal `"rayfish"`), `self_update` (**CON-006**, `SELF_UPDATE_ENABLED` must be `false`), `host_identity` (**CON-007**, curated-token `leak_count` must be `0` across `src/**/*.rs`), `build_tooling_identity` (**CON-008**/RENAME-010, the same curated-token approach for non-Rust build tooling — `justfile`/`contrib/` — that CON-007's `src/**/*.rs` scan can't cover; `unexpected_count` must be `0`), `report_identity` (**CON-009**/RENAME-014, curated `rayfish` product-name tokens on the diagnostic/report + repo surface — `src/**/*.rs` plus `.github/**` and `cliff.toml`; `unexpected_count` must be `0`), `cli_reference_identity` (**CON-010**/RENAME-016..017, a regex — `(?<![.\w-])ray (?=[a-z])` — that keeps the pre-fork `ray <verb>` binary reference out of `src/**/*.rs` **and** the `tests/` harness while the lookbehind skips the KEEP forms `.ray`/`ray-proto`/`ray-mobile`/`rayfish`; `unexpected_count` must be `0`), and `test_harness_identity` (**CON-011**/RENAME-017, curated functional `rayfish` service/config/marker/record tokens in `tests/` — `systemctl … rayfish`, `/etc/rayfish`, `Added by rayfish`, `_rayfish_certgen`, `rayfish/files/1` — kept out of the e2e harness while the `NAMES=(rayfish-*)` cloud-VM labels stay; `unexpected_count` must be `0`).
- **libspec** — spec snapshots are recorded **manually after each commit**: `/home/erik/code/libspec/.venv/bin/libspec link --vcs git --revision <sha> --metadata "..."`. `libspec diff` previews. There is **no `build` command**. Each such link mutates `.libspec/libspec.jsonl`, which is folded into the next commit.

**The loop for a change:** amend/add the requirement class in `spec/design_spec.py` → implement → `python3 reconcile.py` until green → commit (conventional subject; **no authorship trailers of any kind**) → `libspec link` the new sha.

# Product & architecture reference

P2P mesh VPN powered by [iroh](https://iroh.computer). Connects peers by cryptographic identity (EndpointId), not IP address. Dual-stack addressing: stable IPv4 in a **configurable** overlay subnet (default `10.88.0.0/16`, an uncommon 10.x slice that avoids Tailscale's `100.64.0.0/10`; FNV-1a of identity, host-bit width follows the prefix) and stable IPv6 in `200::/7` (blake3 of identity, 120-bit, never rotates).

## Build

```bash
cargo -q build                 # add --features tor for Tor transport, --features otel for OTLP span export
cargo -q check
cargo -q test
cargo -q clippy
cargo bench                    # Criterion microbenchmarks of the per-packet data path (benches/forward.rs)
cargo build --release          # distributable binary at target/release/torpedo
```

The crate splits into a library (`src/lib.rs`, daemon modules as `pub mod`) and a thin binary (`src/main.rs`, the `torpedo` CLI/IPC client, `use rayfish::…` — the library name is kept, see KEEP-ON-PURPOSE). The split lets benchmarks (`benches/`) and integration tests reach the internal data path; `cargo install` builds the binary against the in-package library unchanged.

**justfile:** `just cross`/`just deploy`/`just deploy-dev` identity was fixed (`binary := "torpedo"`, `groupadd torpedo`, `systemctl restart torpedo`) — safe to use. `ray-mobile`/`libray_mobile` (Android crate/artifact name) is a separate, deliberately-undecided item (RENAME-010, out of scope).

## Run

The daemon (`torpedo daemon`) owns the TUN device and iroh endpoint and runs as a system service. CLI commands talk to it over Unix-socket IPC.

```bash
sudo torpedo up                # install+start the service, then activate the VPN
torpedo create [--open] [--name n] [--hostname h] [--subnet CIDR] [--tor]   # closed by default; --open = public. --subnet overrides the default 10.88.0.0/16 for this network. Prints room id (public key)
torpedo join <room-id-or-invite> [--name alias] [--hostname h] [--auto-accept-firewall] [--tor]  # join by room id or one-time invite code; --auto-accept-firewall auto-installs suggested rules (managed node/server)
torpedo leave <net> | nuke <net>   # nuke = publish empty record then leave
torpedo kick <net> <peer>      # coordinator-only (closed networks): remove a member. Prunes it from the roster + approved list, republishes the signed blob, disconnects it mesh-wide. Refused on open networks (target would re-join) and against coordinators/self
torpedo ephemeral <net> <dur|off|show>   # coordinator-only, per-network auto-kick TTL: a member offline longer than <dur> is removed automatically (same teardown as kick). <dur> is Nh/Nd/Nw (1h floor); off disables; show prints it. Off by default, applies to open + closed (only offline peers pruned). Surfaced in status
torpedo hostname <net> <name>  # change hostname on existing network
torpedo status                 # all networks (works without daemon); per-host traffic, member count excludes self. Ends with a `pending` summary of things awaiting the user (firewall suggestions, join requests, file offers, connection requests) each with the command that clears it
torpedo <cmd> --json           # global flag: machine-readable JSON for status/firewall show/files/invite list/requests/admin list/ping/netcheck/identityof (color + spinners off)
torpedo report                 # bundle logs+metrics, open a pre-filled GitHub issue (REPORT_REPO_URL = the torpedo fork)
torpedo ping <peer> [-c N] [-i ms]   # active mesh probe: per-probe RTT + loss + direct/relay path. Sends live echo probes (ControlMsg::Ping/Pong over the mesh connection). -c/--count (default 3), -i/--interval ms (default 1000); --json emits the per-probe array
torpedo netcheck               # local endpoint diagnostics: bound UDP port (+ fixed-forwardable vs ephemeral fallback), home relay + latency, public IPv4/IPv6, UDP reachability (iroh net report). --json
torpedo up [--hostname h] | down   # activate / standby. down takes the data plane (TUN + Magic DNS) offline but stays connected to peers (still online); --hostname sets your default name

torpedo invite <net> [create] [--expires 7d] [--hostname H] [--qr]   # coordinator-only: mint single-use invite; `create` is the default action (bare `torpedo invite <net>` mints one with defaults) but MUST be spelled out to attach any flag — clap will not parse `--hostname`/`--expires`/`--qr`/`--reusable` onto the bare `invite <net>` form. --qr prints a scannable QR; --hostname binds an authoritative name (overrides joiner choice, rejected on collision)
torpedo invite <net> create --reusable [--expires 30d]          # mint a reusable (multi-use, expiring) key for unattended fleets; rides the signed blob, no hostname binding. Servers: torpedo join <key> --hostname H --auto-accept-firewall
torpedo invite <net> list|revoke <id>          # list / revoke invites (reusable keys tagged; revoke propagates via the blob)
torpedo requests <net>         # coordinator-only: peers awaiting live approval
torpedo accept <net> <id> | deny <net> <id>    # admit / reject a pending join request
torpedo connect <contact-id> [--hostname h]    # request a direct 2-peer connection by the peer's contact id (no room id/invite); blocks as pending until they approve
torpedo connections [approve <id>]             # list incoming connect requests (default) / approve one → mints a 2-peer network with the requester pre-approved
torpedo contact [id|rotate]    # print (default) or rotate your shareable contact id (also shown at the top of status)
torpedo admin <net> add <id> | list            # coordinator-only: grant the network key (co-coordinator) / list key-holders. <id> is the target's short id as shown in `torpedo status` (its endpoint-id prefix) — unlike `--peer` elsewhere, hostname is NOT accepted and resolves with "could not resolve identity"
torpedo firewall show|default|add|remove|on|off ...        # per-device local firewall (userspace, separate from and layered on top of the host/kernel firewall, both must allow a packet). Default posture: inbound TCP/UDP denied, inbound ICMP allowed, outbound allowed. `firewall default allow|deny` sets the inbound default. `firewall off` (alias disable) is a global kill switch: config.disabled = true makes evaluate_packet allow every packet (rules/defaults bypassed; the ingress anti-spoof check in forward::evaluate_inbound still runs, it is upstream); `firewall on` (alias enable) restores enforcing. Surfaced in firewall show. --port/-P takes a single port, a start-end range, or a comma list (80,443, 22,8000-9000) that expands to one rule per item. --peer accepts any peer identifier (hostname, mesh IPv4/IPv6, short id, full endpoint id, or a paired user identity), resolved daemon-side; an `in` rule is keyed on the peer's user identity (matches every device of that user), an `out` rule on the device id
torpedo firewall reject on|off # "fail fast" REJECT mode (opt-in, default off). On = a denied packet gets a TCP RST / ICMP-unreachable reply (both directions) so the initiator fails immediately ("connection refused") instead of hanging; off = silent drop (stealthy). Surfaced in firewall show
torpedo apply <spec> [--prune] [--dry-run] [--invite-missing] [--example]   # declarative deploy (YAML only): create closed nets + suggest firewall + report membership gap. Optional top-level aliases: (name → identity) and groups: (name → [alias|hostname]) are coordinator-side shorthand, expanded client-side into a plain hostname-keyed firewall (never reach the blob). An alias names a user (its user_identity, or device endpoint id if unpaired) and expands to all that user's currently-joined device hostnames; a group is a named set of aliases/literal hostnames usable as a subject or peer. Precedence subject/peer = group → alias → literal. Aliases resolve only for already-joined members (warn + skip if zero joined); literal hostnames remain the pre-join identifier
torpedo identityof <net> <host> [--json]   # print a host's identity string to paste into a spec's aliases: (user_identity if paired, else device endpoint id). Open read; errors if the host isn't currently joined
torpedo alias <net> set <key> <alias> | list | rm <alias>   # node-local, per-network alias for a user. key = an identity string (from identityof) or a currently-joined hostname (resolved locally to its identity). Display-only: shown inline in status as host.net.ray [alias], and seeds apply's aliases: map (spec wins on name conflict). Never rides the signed blob. set/rm mutate (root/operator); list is an open read (--json)
torpedo firewall suggest <net> --subject H [--allow peer:proto:ports] [--deny peer:proto:ports]  # coordinator-only: suggest rules on any network (rides the signed blob). Subject/peer * = all hosts / any peer. --allow/--deny value is [peer:]proto:ports — the peer: prefix is optional, so a bare tcp:22 (or icmp) means "any peer". Token grammar: proto:ports (tcp:22, udp:53, tcp:*, any:*) or bare proto (icmp, any, tcp). Additive — each token becomes one allow/deny rule; an allow-list relies on the node's own inbound default-deny (no catch-all synthesized), denies-only ⇒ blacklist
torpedo firewall pending <net> | accept <net> | deny <net>  # review/accept/discard queued suggested rules. On a TTY, pending is an interactive picker (↑↓ · enter accept · d deny · a all · q done); piped/--json falls back to a static table
torpedo firewall auto-accept <net> on|off  # toggle this node's auto-install of suggested rules for a network (on = install current queue)
torpedo firewall ssh on|off | allow <net> <peer> [--user u,...] | deny <net> <peer> | show [<net>]   # embedded mesh SSH (Tailscale-style, no SSH keys). ssh on runs an SSH server on this node's mesh IPs:22 (seeds an allow in tcp:22 passthrough, origin Ssh) and, when no network has any ssh_allow entry yet, nudges the user to authorize a peer (admits nobody until an allow entry exists); allow <net> <peer> authorizes a peer (hostname/mesh-ip/short-id, or * = any peer) to log in. --user/-u (comma list) restricts local unix users: omitted ⇒ any non-root user (secure default), named accounts ⇒ only those, * ⇒ any user incl. root; setting a peer's rule replaces its user list. Per-user check is by uid (a uid-0 account is blocked unless root is granted). Connect with a stock client (ssh user@host.ray); the peer is identified by its mesh identity (proven by the QUIC link → auth_none is the gate). Global toggle in settings.toml (ssh_enabled); per-network allow list in networks/<name>.toml (ssh_allow, a list of SshRule { peer, users })
torpedo mdns on|off            # local peer discovery (default on)
torpedo config [get [key] | set <key> <value> [--replace] | unset <key>]   # global daemon settings; keys: relay, discovery-dns, dns-upstreams, subnet, magic-dns. relay/discovery values are comma lists of presets (rayfish/n0), URLs, or IPv4s (default augments n0; --replace swaps them out; n0/empty resets). `subnet` takes a single CIDR (e.g. 10.88.0.0/16; empty resets to default). `magic-dns` is `off|auto|direct` (default `auto`): `auto` uses a clean split-DNS backend if present and otherwise leaves `/etc/resolv.conf` untouched (so `.ray` will not resolve locally — reach peers by mesh IP from `torpedo status`); `direct` opts into the `/etc/resolv.conf` takeover for `.ray` on a minimal host (guarded against foreign-VPN DNS loops); `off` never configures system DNS. Written client-side to settings.toml; all apply on `sudo torpedo restart`
torpedo send <file> <peer>     # file sharing; torpedo files [accept <id> [--output dir]]
torpedo files auto-accept <net> on|off   # auto-accept file offers from your own paired devices on this network (own-devices-only). on also drains already-queued own-device offers. Off by default. Also settable at join via torpedo join --auto-accept-files
torpedo files download-dir <path> | download-user <user>   # where auto-accepted (own-device) files land: an absolute dir, or a user whose ~/Downloads receives them (files chowned to that user). Precedence: download-dir (owner = download-user, else the dir's owner) -> download-user's ~/Downloads -> operator's ~/Downloads -> if none, the offer stays queued. No arg shows the value; --clear unsets. Root on Linux (settings.toml), applied live
torpedo pair [<ticket>|backup|restore <code>]              # multi-device identity
torpedo pair list                          # list your paired devices (enumerated from rosters; --json)
torpedo pair backup [--1password [--vault V] [--item T]]   # encrypted key backup; --1password stores the enc1 blob in 1Password (op CLI)
torpedo pair restore [<code>|--1password [--vault V] [--item T]]   # restore from a code or from 1Password
torpedo unpair <device>        # primary-only: revoke a paired device. Bumps this user's cert generation and publishes the new floor to pkarr (_torpedo_certgen, a single signed number) that every peer consults; re-issues fresh certs to devices you keep (auto-pushed over the mesh; offline keepers refresh on reconnect), drops the target from your networks + local own-device treatment, best-effort tells it to wipe its cert. Enforced verifier-side
torpedo completions <shell>
torpedo version | torpedo --version | torpedo -V        # print the compiled version + git sha
```

> **Self-update is DISABLED in this fork.** The `update` / `auto-update` machinery still compiles (`torpedo update`, `torpedo auto-update on|off`, `sudo torpedo install --auto-update`) but is neutralized behind `update::SELF_UPDATE_ENABLED = false` (**CON-006**), because the fork publishes no releases and `REPO_SLUG` still points at upstream `rayfish/rayfish`. Do not re-enable it. The descriptions below are retained for context only.

**Privilege & access (Tailscale operator model):** the always-root daemon does privileged work; clients are unprivileged. The IPC socket is mode `0666`; authority comes from a per-request `SO_PEERCRED` UID check in `MeshManager::check_authorized()`, not socket permissions. Reads (`status`, `*… show`, `files`) are open to any local user; mutating commands need root or the configured `operator_uid`; `set-operator` is root-only. Only `install`, `restart`, `uninstall`, `start`, `stop`, `set-operator`, and `daemon` need `sudo`; everything else (incl. `up`/`down`) is IPC. `torpedo up`/`install` auto-grant operator to `$SUDO_USER`.

```bash
sudo torpedo install [--auto-update] | restart | uninstall  # manage the service unit/plist (--auto-update is a no-op while self-update is disabled)
sudo torpedo start | stop                    # start / stop the service. stop = fully offline (closes peer connections); start = back online
sudo torpedo set-operator <user>             # authorize a user to run torpedo without sudo
```

## Architecture

```
App → TUN (10.88.x.x / 200::x) → torpedo → iroh QUIC datagrams → peer
```

One iroh Endpoint and TUN device are shared across all networks. Each network gets its own ALPN (`torpedo/net/<version>/<pubkey-prefix>`); the `ProtocolRouter` dispatches incoming connections by ALPN to per-network handlers. The leading `<version>` (`transport::MESH_PROTOCOL_VERSION`) makes the ALPN the **mesh protocol-version gate**: iroh negotiates the ALPN during the QUIC handshake, so peers on different mesh versions share no common ALPN and cannot connect — no in-band version handshake exists. Bumping the constant on a breaking mesh change severs old peers automatically (likewise the versioned `connect`/`files`/`pair` ALPNs gate their own protocols).

### Modules

- `src/main.rs` — thin clap CLI + IPC client: the `Cli`/`Command` definitions, `main` dispatch, tracing/panic plumbing, and shared presentation helpers. `torpedo daemon` (hidden) runs the foreground daemon loop. `build.rs` stamps the git short SHA (see Self-update). The command handlers are split into `src/cli/` — one file per domain (`network`, `status`, `invite`, `connect`, `firewall`, `files`, `pair`, `service` — incl. `cmd_up`/`install_and_start_service`/`cmd_install`/`cmd_restart`/`cmd_uninstall_service`/`cmd_set_operator`, `update`). Each opens with `use crate::*;`; `src/cli/mod.rs` flattens them with `pub(crate) use <m>::*` (private `mod`s so only contents re-export — avoids clashing with the `use rayfish::{firewall, …}` aliases), and `main.rs` does `use cli::*`, so every handler resolves the others (and the shared helpers) by name. Handlers and the consts/enums they cross are `pub(crate)`.
- `src/daemon/` — daemon process, split into a directory module. The core type is **`MeshManager`**: it holds the endpoint, TUN, `PeerTable`, networks map, blob store, firewall, and `ProtocolRouter`, plus three **domain satellites** it owns as fields — `DnsManager` (`dns_manager.rs`: `.ray` naming tables, resolver, OS-DNS configure/revert), `FileService` (`file_service.rs`: pending file offers, pairing secret, and the `FILES_ALPN`/`PAIR_ALPN` accept arms), and `ConnectService` (`connect_service.rs`: the `torpedo connect` pending/approved/outgoing maps and the `CONNECT_ALPN` accept arm). The satellites own their own state + ALPN accept arms; `ProtocolRouter` holds `Arc` clones of `FileService`/`ConnectService` and delegates those arms to them. `mod.rs` holds the core type definitions (`MeshManager`/`NetworkState`/`NetworkHandle`/`MeshCtx`), the `impl MeshManager` IPC dispatcher (`handle_request`, `check_authorized`, `set_operator`, `set_hostname`, `mesh_ctx`, `register_coordinator_handler`/`promote_to_coordinator`, `resolve_short_id_any_network`), the roster-projection + control-message helpers, and unit tests. The bulk lives in `src/daemon/mesh/` — each file opens with `use super::super::*;` and is re-exported (`pub(crate) use mesh::*`) so callers reach the moved free functions by bare name. Per-domain IPC ops are `impl MeshManager` method-modules: `create_join`, `runtime`, `diagnostics`, `invite`, `admin`, `connect`, `files`, `firewall`. Mesh machinery: `accept.rs` (`CoordinatorAcceptState`/`MemberAcceptState`/`AcceptHandler`/`MeshProtocol`/`ProtocolRouter`; closed-network pending-join queue bounded by `MAX_PENDING_JOINS`, oldest-eviction via `evict_oldest_pending`), `join.rs` (`join_mesh_shared` + `JoinParams` + `spawn_reconnect_loop`), `bootstrap.rs` (`run_daemon`, `build_daemon`, `serve_ipc`/`handle_ipc_client`, metrics/mDNS wiring). The former `background.rs` is five files: `publish.rs` (DHT network/contact/lazy publishers + `update_snapshot_and_publish`), `reconverge.rs` (`fetch_verified_blob`/`reconverge_and_apply`, group poller, `apply_suggested_firewall`, `apply_roster_to_dns`), `coordinator.rs` (coordinator control reader, peer cleanup, invite-gossip helpers), `select.rs` (pure decision helpers — `coordinator_dial_order`, `gossip_targets`, `pick_first_welcome`, `choose_path_index`, `persisted_roster`), `rename.rs` (pending-hostname drain + `rename_satisfied`). `create_join.rs`'s `join_network_inner` reads as three phases — `resolve_and_fetch_blob` → `dial_fresh_join`/`dial_reconnect` → `finalize_join` — with per-join inputs bundled in `JoinContext`. **Shared-handle bundling:** `MeshCtx` groups the daemon-wide cheap-`Clone` handles (identity, `PeerTable`, `tun_tx`, stats, blob store, firewall, hostname/reverse DNS tables, `DeviceUserMap`) into one value built by `MeshManager::mesh_ctx()`, embedded in the accept states and passed as one argument to spawn/reconverge free functions. `MeshCtx::forward_ctx(disconnect_tx, token)` builds a peer's `forward::ForwardCtx`. Admission gate: `CoordinatorAcceptState::handle_connection` → `admit_peer` (open / valid invite / pre-approved) vs queue-as-pending (closed). Diagnostics `ping()`/`netcheck()` are open reads. **Coordinator setup is shared:** `create_network_inner` and `restore_coordinator_network` both go through `seal_and_publish` + `spawn_coordinator_background_tasks`.
- `src/ipc.rs` — thin shim; the `IpcMessage` enum, `MsgpackCodec` (length-prefixed msgpack over Unix socket), and socket helpers live in `ray-proto`. Socket at `/var/run/torpedo/torpedo.sock`.
- `src/identity.rs` — persistent Ed25519 keypair (`<config_dir>/secret_key`, `0600`); device certs (`create/store/load_device_cert`). Resolves its dir via `config::config_dir()`.
- `src/onepassword.rs` — `op` CLI wrapper for `torpedo pair backup/restore --1password`. Transports the existing `enc1…` encrypted backup blob to/from a 1Password item; CLI-side only.
- `src/invite.rs` — coordinator-only **single-use** invite ledger (`<config_dir>/invites/<network>.toml`, `0600`, atomic): `InviteStore` (`mint`/`redeem`/`revoke`/`list`/`restore`/`record_shared`/`burn_by_hash`; single-use + expiry; only the blake3 hash persisted). `encode_invite_code`/`decode_invite_code` = `bs58(network_pubkey(32) || coordinator(32) || secret(16))`. **Reusable keys live in the signed blob** (`membership::ReusableKey`), not here.
- `src/membership.rs` — `IdentityProvider`, IPv4/IPv6 derivation, `MemberList`/`ApprovedList`, `GroupBlob { members, approved, suggested_firewall, name, reusable_keys }` with canonical msgpack + blake3 hashing. **Subnet is configurable:** `default_subnet()` returns `10.88.0.0/16` (an uncommon 10.x slice avoiding Tailscale's `100.64.0.0/10`, SUBNET-011); the authoritative subnet rides the `GroupBlob`/config, host-bit width follows the prefix, and IP derivation is subnet-relative. `assign_ip` picks the lowest free collision index at admission; `validate_member`/`validate_approved`/`validate_no_duplicate_ips`/`resolve_ip_tiebreak` keep rosters consistent. `SuggestedFirewall`/`HostSuggestions` live in `ray-proto` (`policy.rs`).
- `src/transport.rs` — iroh endpoint setup, per-network ALPN (`network_alpn` → `torpedo/net/<MESH_PROTOCOL_VERSION>/<prefix>`); identity-level `CONNECT_ALPN` (`torpedo/connect/1`); `FILES_ALPN` (`torpedo/files/1`); optional Tor transport (`tor` feature). The shared endpoint binds a **fixed UDP port** `TORPEDO_LISTEN_PORT` (**43737**), stable across restarts and manually port-forwardable for guaranteed direct reachability (iroh still does NAT traversal/UPnP/PCP, discovery, and relay fallback). If the port is in use the daemon warns and falls back to an ephemeral port. **Custom relay/discovery overrides:** `bind_endpoint` keeps `Endpoint::builder(presets::N0)` and conditionally overrides from `config::ServerOverride` — `build_relay_mode` swaps relays, `apply_discovery` stacks pkarr publisher/resolver per configured discovery URL. No-ops when unset. Set via `torpedo config set relay|discovery-dns ...`.
- `src/tun.rs` — async dual-stack TUN (IPv4 + IPv6 /128), split into `TunReader`/`TunWriter`. `configure_ipv6()` assigns the TUN's own IPv6 at creation. `route_peer_range()` installs the `200::/7` peer-range route **after link-up** (from `MeshManager::activate()` post-`set_link_up`). `route_self_loopback(v4, v6)` installs `-host` routes for our own addresses via `lo0` (macOS only) so self-traffic is answered locally. Idempotent across `up`/`down`.
- `src/forward.rs` — TUN ↔ peer forwarding via dual-stack routing lookup; firewall enforcement; labeled drop counters; resolves transport keys to user identities via `DeviceUserMap`. `spawn_peer_reader` takes its data-plane handles as a single `ForwardCtx` bundle. `run_mesh` intercepts UDP packets destined for the subnet-derived magic resolver address (port 53) and answers them in-daemon via the `dns_resolver::Resolver`, so Magic DNS never binds the host's port 53. On a firewall deny with `reject` mode on it emits a `reject::build_reject` reply.
- `src/reject.rs` — "fail fast" REJECT reply builder (opt-in via `torpedo firewall reject on`). Synthesizes a TCP RST for denied TCP, or an ICMP/ICMPv6 destination-unreachable otherwise, src/dst swapped. Hand-rolled checksums. Returns `None` to keep silent-dropping loop-risk cases.
- `src/ssh.rs` — embedded mesh SSH server, Tailscale-style: a stock `ssh user@host.ray` lands here over the already-authenticated mesh link, so `auth_none` is the gate. `russh` binds `:SSH_LISTEN_PORT` (30022) and `forward.rs` runs a userspace `:22`↔`:30022` NAT (leaves host sshd untouched, gated by `set_ssh_nat_active`). Admits iff the peer (or `*`) is in a shared network's `ssh_allow` **and** the requested unix user passes `UserPolicy::permits` (uid-checked). Drops privileges in `pre_exec`. Host key reuses the machine's real OpenSSH ed25519 host key (via `sshd -T`), falling back to a persisted generated key. Live per-network allow lists (`SshAuthz`) swapped atomically.
- `src/dht.rs` — one pkarr record per network (blob hash + seed peers + `m,<mesh-version>`); only the coordinator (per-network secret key) can publish. Plus a per-user **contact record** (`_torpedo_contact`, signed by the contact key) for `torpedo connect`; a TTL/2 active-gated publisher keeps it fresh. The pkarr relay URL defaults to `dns.iroh.link/pkarr` but follows the `discovery-dns` config when set. Also a per-user **cert-generation floor** (`_torpedo_certgen`, signed by the user's own key) for `torpedo unpair`.
- `src/revocation.rs` — `RevocationCache`, the daemon-wide `user_identity → generation floor` cache, plus the pure `cert_decision` helper. **Fail-open when unseen (floor 0), monotonic once seen.** See the Unpair key flow.
- `src/control.rs` — length-prefixed msgpack control protocol over QUIC streams (`JoinRequest`, `JoinPending`, Welcome, MemberApproved, MeshHello, BlobUpdated, `AdminGrant`, `InviteShare`, `InviteUsed`, `Ping`/`Pong`, `CertRefresh`/`Unpaired`, …); `DeviceCert`, `PairMsg`. `ConnectMsg` is a separate enum for the `torpedo connect` handshake over `CONNECT_ALPN`. All triggers pass through the `ControlGate` rate limit.
- `src/peers.rs` — `PeerTable` (dual v4/v6 DashMaps), `DeviceUserMap`. A peer keeps one virtual IP across every network it joins, so each `PeerEntry` holds a *set* of connections; `lookup_v4/v6` return a `PeerRoute`. `remove_peer_from_network()` drops one network's route, `remove()` drops it everywhere.
- `src/config.rs` — config storage. **Sharded, atomic, per-network:** globals in `settings.toml` (`mdns_enabled`, `operator_uid`, `default_hostname`, `contact_secret_key`, the `ServerOverride` settings `relay`/`discovery_dns`/`dns_upstreams`, and the overlay `subnet`), each network in `networks/<name>.toml` (per-network secret/public key, `my_hostname`, `pending_hostname`, `group_mode`, `auto_accept_firewall`, `admins`, `direct`, per-network subnet). Writes go through `write_file` (temp + atomic `rename`) and are targeted. `load()` assembles the in-memory `AppConfig`, running a one-time `migrate_legacy`. **Location:** `config_dir()` is `/etc/torpedo` on Linux (system service), `~/.config/torpedo` on macOS; dirs `0750 root:torpedo`, secret-bearing files `0600 root:root`, non-secret `0640 root:torpedo` (the `torpedo` group is created on Linux install). **`ServerOverride` + subnet** carry pure resolver/validator helpers; `config_set`/`config_get` back the `torpedo config` CLI (subnet takes a single CIDR; n0/empty resets server keys; unknown key/bad URL/IP/CIDR rejected before persist). **The relay preset keyword `"rayfish"` is retained (CON-001) — do not rename.**
- `src/apply.rs` — declarative deploy spec for `torpedo apply`: `DeploySpec { aliases, groups, networks: BTreeMap<network, SuggestedFirewall> }`. **YAML only** (keys lowercased by the `config` crate). `expand_firewall` is the pure client-side expansion (precedence group → alias → literal; `*` untouched). `EXAMPLE_SPEC` printed by `--example`. Orchestrator in `main::ipc_apply`; `cmd_identityof` backs `torpedo identityof`.
- `src/firewall.rs` — per-device firewall (direction/proto/port/peer + optional arrival-`network`), `ArcSwap` for lock-free reads, dual-stack packet parsing; `firewall.toml`. Direction-aware defaults + seeded `allow in icmp` rule. `RuleOrigin` (`Local` | `Network(net)`) records provenance so reconvergence replaces the `Network(net)` set without touching `Local` rules. `materialize_suggestions` builds a subject's inbound rules additively.
- `src/dns.rs` — Magic DNS responder for the **`.ray` TLD** (A/AAAA/PTR/SOA for `*.ray`; the naming scheme is kept, see KEEP-ON-PURPOSE); reached via the subnet-derived magic IP (`dns::magic_dns_v4`, e.g. `10.88.100.53` on the default subnet) routed through the TUN — no host-level port 53 bind. `sync_network_hostnames` rebuilds a network's forward+reverse entries from its member roster on every roster update.
- `src/dns_config.rs` — OS DNS config (`DnsConfigurator` trait). Points the OS at the subnet-derived magic IP so `.ray` queries are intercepted. Linux detection chain: systemd-resolved D-Bus → NetworkManager D-Bus → resolvectl → resolvconf → `/etc/resolv.conf` takeover. **Host artifacts renamed to torpedo (RENAME-006):** the NM drop-in `/etc/NetworkManager/conf.d/torpedo-dns.conf`, the resolv.conf marker `# Added by torpedo`, the backup suffix `.before-torpedo`, resolvconf tag `tun-torpedo`, and the macOS SCDynamicStore key `State:/Network/Service/torpedo/DNS` / client name `"torpedo"` — so torpedo and rayfish do not collide on one host. **Direct-mode anti-trample** (inotify re-assert + NM `dns=none` drop-in) and **crash safety** (`emergency_restore_resolv_conf()` in the panic hook) unchanged. Both the NM drop-in and the resolv.conf backup are marker-guarded so we never touch an operator's own config.

### Key flows

- **Create:** generate per-network `SecretKey` → derive addresses from the network's subnet (default `10.88.0.0/16`, or `--subnet`) → build initial `GroupBlob` → publish blob + signed pkarr record → persist keys + `group_mode` + subnet → print public key as the room id. Closed (`Restricted`) by default; `--open` for public.
- **Access modes & admission:** the room id (network public key) is a published discovery key, **never** an admission credential. **Open** networks auto-admit any peer that reaches a coordinator. **Closed** networks gate three ways: a one-time **invite** (coordinator-only local ledger, gossiped via `InviteShare`/`InviteUsed`); a **reusable key** (hash rides the signed `GroupBlob.reusable_keys`); or **live approval** (unknown peer queued in `NetworkState.pending`, surfaced via `torpedo requests`, admitted with `torpedo accept`). The handler is `CoordinatorAcceptState`, run by **any node holding the network key**. The admitting coordinator assigns the joiner's IPv4 via `assign_ip`.
- **Join handshake:** resolve pkarr record → fetch + verify `GroupBlob` → dial in `coordinator_dial_order` (invite-pinned minter first, then other `is_coordinator` members) until one replies `Welcome` → send `JoinRequest { invite_secret? }` first → coordinator replies `Welcome`, `JoinPending`, or `JoinDenied`. `torpedo join <reusable-key> --hostname H --auto-accept-firewall` is the unattended-server path. Then connect to other members with `MeshHello` and poll pkarr for blob updates.
- **Gatekeeper:** any coordinator (any network-key holder) can approve identities and broadcast `MemberApproved`; admitting a fresh joiner survives any single coordinator being offline. The coordinator need not be online for *member* reconnects at all.
- **DHT (single-record):** one pkarr record per network signed by the per-network secret key. The pkarr address *is* the network public key, so records can't be spoofed. `spawn_group_poller()` refetches the blob every 60s when the hash changes.
- **Reachability model (segmentation-first):** a network is a reachability boundary — two peers exchange packets iff they share ≥1 network (a QUIC connection only exists within a shared network). Coarse access is the network split; the per-device firewall is the fine-grained layer. Declarative provisioning is `torpedo apply`.
- **Firewall (local + coordinator suggestions):** per-device, first-match-wins, persisted in `firewall.toml`, with a stateful conntrack so return traffic for outbound flows passes under a deny default. **Secure-by-default inbound** (`default_inbound` Deny, `default_outbound` Allow); ICMP-allow is the seeded, removable `allow in icmp` rule. On **any** network the coordinator can **suggest** rules — advisory, riding the signed `GroupBlob`. Consent is **per-node, per-network**: auto-accept or manual `torpedo firewall accept|deny`. Rules re-materialize on every verified reconverge; `Local` rules are never touched. Suggestions come only from the verified blob, never from a control message. **Fail-fast (REJECT) mode** and the **global kill switch** (`torpedo firewall on|off`) are opt-in per-device toggles.
- **Multiple admins = shared network key.** `torpedo admin <net> add <id>` grants the key to a member over the authenticated mesh ALPN (`AdminGrant`), making it a co-coordinator that can publish the signed blob, suggest firewall rules, and admit fresh joiners. The grantee calls `promote_to_coordinator` on receipt.
- **Declarative apply (`torpedo apply`):** reconcile networks against a YAML spec (a `networks:` map of `<name> → SuggestedFirewall`, plus optional `aliases:`/`groups:`). `main::ipc_apply` fetches `Status` once, expands each network's firewall client-side, then per network: `Create` (closed) if absent, then publishes the expanded firewall block as suggestions (idempotent). `--prune` drops out-of-band suggestions; `--dry-run` echoes the expanded spec; `--example` prints a template; `--invite-missing` mints invites for the membership gap. **Aliases & groups are pure spec sugar (never reach the blob).** Literal hostnames stay the provision-ahead identifier (invite-bound, authoritative).
- **Direct connections (`torpedo connect`):** a friend-request flow linking two peers with no shared room id or invite. Each node has a rotatable **contact key** published to pkarr while active (`_torpedo_contact` = `contact_pubkey → endpoint`) and advertised over `CONNECT_ALPN`. `torpedo connect <contact-id>` resolves → dials → `ConnectMsg::Request`; the recipient queues it and replies `Pending`; `torpedo connections approve <id>` mints a restricted 2-peer network with the requester pre-approved. `torpedo status` shows role `[direct]` and hides the room id. If both peers connect and approve at once, only the higher `endpoint.id()` mints.
- **File sharing:** `torpedo send` adds the file to iroh-blobs and sends a `FileOffer` over `FILES_ALPN`; receiver queues it; `torpedo files accept` fetches + verifies by hash. **Auto-accept (own devices only):** `try_auto_accept_file` accepts only when the sender's resolved `user_identity` equals ours **and** the sender is a member of some network with the flag on. Files land at `resolve_download_target()` (download-dir → download-user's ~/Downloads → operator's ~/Downloads → else queued).
- **Pairing:** primary issues a ticket (`bs58(endpoint_id || secret)`) over `PAIR_ALPN`; secondary authenticates and receives a `DeviceCert` binding its transport key to the primary's user identity. Backup/restore encrypts the identity key (argon2 + chacha20poly1305) into an `enc1…` base58 blob. `--1password` transports that blob to/from a 1Password item (blob stays password-encrypted). All `op` calls are CLI-side, never from the root daemon.
- **Unpair / device-cert revocation (`torpedo unpair`):** revocation is a **generation floor** verifiers consult, signed by the **user identity**. The primary publishes a signed `_torpedo_certgen` pkarr record carrying a monotonic generation; any peer that would trust a cert checks it via `revocation::cert_decision` against the daemon-wide `RevocationCache` (fail-open when unseen, monotonic once seen). Rotation bumps `cert_generation`, adds the device to the local never-published `revoked_devices` set, publishes the new floor, re-issues fresh certs to kept devices (`CertRefresh`), removes the target from every network it coordinates, severs live links (`KICK_CODE`), and best-effort sends `Unpaired`. Only the primary can unpair (restore from `pair backup` if lost); the signed floor is the guarantee.
- **Hostname change:** `torpedo hostname` propagates immediately and is coordinator-authoritative. A member's rename re-sends `MeshHello`; the coordinator resolves collisions, updates roster + DNS, republishes the blob, broadcasts a payload-free `MemberSync` trigger. **Reliable delivery:** the rename is persisted as `config.pending_hostname` (durable intent) re-sent on every reconnect via `drain_pending_rename` until the signed roster confirms it (`rename_satisfied`). `apply_roster_to_dns` is pending-aware. Admission hostname authority follows the **invite binding** (`torpedo invite <net> create --hostname <h>`): an invite-bound hostname is assigned exactly, a clash with a different identity is rejected.
- **Reconnection:** per-peer reader detects drop → coordinator removes the dead peer; joiner reconnects with exponential backoff (1s–30s) then re-sends `MeshHello`.
- **Control-plane abuse defense:** both control read loops gate each connection with a per-task token bucket (`ratelimit::ControlGate`). Over-budget messages are dropped; a sustained flood trips `Verdict::Close` (`forward::ABUSE_CODE`). `MemberSync`/`BlobUpdated` only `notify_one()` a per-network debounced reconverge worker (~300ms coalesce, single-in-flight).
- **Leave:** `torpedo leave` gracefully closes its connections with `forward::LEAVE_CODE`. Peers see `DisconnectEvent.intentional = true`; the coordinator prunes the member, republishes, and broadcasts a payload-free `MemberSync`. A plain timeout/reset is *not* intentional.
- **Kick:** `torpedo kick <net> <peer>` is coordinator-authoritative, closed-networks only. `kick_member` removes it from the roster + approved, drops DNS, republishes, broadcasts `MemberSync`, and closes its own link(s) with `KICK_CODE`. **Mesh-wide teardown is roster-driven:** `prune_departed_peers` closes + unroutes any live connection whose peer is no longer in the applied roster, recording it in `MeshCtx.pruned_peers` so the reconnect loop skips the re-dial.
- **Ephemeral auto-kick:** an opt-in per-network TTL (`torpedo ephemeral <net> <dur>`, off by default, 1h floor). `spawn_stale_member_pruner` (coordinator-only) sweeps every ~30min via the pure `should_prune` (skips coordinators, self, connected peers, and members with no `last_seen`). The staleness clock is `Member.last_seen` in the signed `GroupBlob`. Only offline peers are removed, so it applies to open + closed alike.
- **up/down (data plane) vs start/stop (whole daemon):** the daemon connects every saved network at startup (control plane) and keeps those connections for its lifetime. `activate()`/`deactivate()` toggle only the **data plane** (TUN link, routes, Magic DNS, inbound forward gate). `torpedo down` is standby (still online to peers, no traffic, no `.ray` resolution); `torpedo up` is near-instant. `sudo torpedo stop`/`start` is fully offline/online.
- **Report:** `torpedo report` → daemon `build_report()` gathers sysinfo + a metrics snapshot + the *sanitized* `StatusResponse` (no secret keys) + recent log files, writes a `.tgz` to `/tmp`, chowns it to the calling UID. The CLI prints the path and opens a pre-filled GitHub issue at `REPORT_REPO_URL` (the torpedo fork).
- **Self-update / auto-update — DISABLED (CON-006):** the machinery compiles but every path is short-circuited by `update::SELF_UPDATE_ENABLED = false`. `REPO_SLUG` stays `rayfish/rayfish`. Do not re-enable without an explicit decision, a fork release pipeline, and a repointed `REPO_SLUG`.
- **Tor (optional):** `--tor` adds `TorCustomTransport` alongside relay; onion address derived from the iroh `SecretKey`. Needs a Tor daemon (`ControlPort 9051`).

## Conventions

- Use `cargo -q` for all cargo commands; `tracing` for logging. `main::init_tracing` composes layers (console + file + optional OTLP) with **split filters**: console/CLI stays at `info`, rolling daily files under `logdir::log_dir()` capture our crate at `debug` (`info,rayfish=debug` — the crate name `rayfish` is kept, so the filter target is too; dependencies stay at `info`). `RUST_LOG` overrides both. Returns a `LogGuard` that must stay alive for the process.
- Tracing carries spans: network lifecycle handlers use `#[tracing::instrument]`; the per-peer reader + reconnect loop wrap tasks in `info_span!("peer"/"reconnect", …)` so report-bundle logs are correlatable.
- `otel` feature (off by default): a `tracing-opentelemetry` layer exporting spans over OTLP/HTTP, active only when `OTEL_EXPORTER_OTLP_ENDPOINT` is set; flushed on shutdown via `LogGuard::drop`.
- Panics are fail-fast in the daemon: `main::install_panic_hook` (set only for `torpedo daemon`) records the panic, appends it to `panic.log`, restores DNS via `dns_config::emergency_restore_resolv_conf()`, then `std::process::abort()`. The service unit restarts it; `panic.log` is bundled by `torpedo report`.
- Never share I/O resources (TUN, sockets, streams) behind a Mutex — split into read/write halves. Avoid Mutex generally: prefer channels, atomics, or `RwLock`/`ArcSwap`.
- CLI subcommands carry short `visible_alias`es (clap): `create`→`new`, `leave`→`rm`, `status`→`st`/`ls`, `version`→`ver`; action verbs `list`→`ls`, `remove`→`rm`/`del`, `show`→`ls`/`list`, `add`→`a`, `revoke`→`rm`, `approve`→`ok`. Aliases must be unique within each `#[derive(Subcommand)]` enum.
- ALPN per network: `torpedo/net/<version>/<pubkey-prefix>` (`MESH_PROTOCOL_VERSION` then first 16 hex chars). File ALPN `torpedo/files/1`, pairing ALPN `torpedo/pair/1`, connect ALPN `torpedo/connect/1`. **The version segment in every ALPN is that protocol's compatibility gate.** Each protocol versions independently — bump the relevant version in the same change that breaks its wire format. iroh negotiates the ALPN at the QUIC handshake, so peers on different versions share no common ALPN and can't connect. `torpedo join` pre-checks the coordinator's signed `m,<mesh-version>` and bails with a precise message; `transport::connect_to_peer_with_alpn` maps an ALPN-mismatch connect error to an "incompatible version" hint.
- TUN MTU 1280 (IPv6 minimum link MTU, RFC 8200 §5; matches WireGuard/Tailscale). Wire format (control + IPC): 4-byte BE length + msgpack body.
- Room id = per-network public key string (discovery only). On a closed network, joining needs a one-time invite or operator approval; on an open network the room id alone admits. Invite code = `bs58(pubkey || coordinator || secret)`. Local aliases (adjective-noun-noun) are display-only.
- Config under `config::config_dir()` (`/etc/torpedo` on Linux, `~/.config/torpedo` on macOS): `secret_key`, `device_cert`, `settings.toml`, `networks/<name>.toml` (one per network), `firewall.toml`, `invites/<network>.toml` (coordinator-only). Pre-migration installs auto-split the old `networks.toml`. On Linux the tree is `root:torpedo`; secret-bearing files are `0600 root:root`. CLI commands that write identity directly (e.g. `torpedo pair restore`) need root on Linux since the tree is under `/etc`.
- Keep commit subjects conventional (`feat`/`fix`/`docs`/`style`/`ci`/…) so git-cliff can render release notes. **Commit messages contain ONLY a description of what was done — never an authorship trailer (`Co-Authored-By`, `Author`, etc.) of any kind.**
- Always update docs (this `AGENTS.md`, `README.md`) after finishing a feature or significant change. And record the change in the spec-first workflow: amend/add the requirement in `spec/design_spec.py`, keep `reconcile.py` green, and `libspec link` the commit.
- Keep `CHANGELOG.md` current as part of every change (`## [Unreleased]`, Keep a Changelog format: `Added`/`Changed`/`Fixed`/`Performance`), describing behavior from the user's perspective. Skip pure-internal churn with no user-visible effect.
